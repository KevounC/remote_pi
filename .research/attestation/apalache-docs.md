# Apalache docs attestation

1. Apalache documentation describes Apalache as a symbolic model checker for TLA+, an alternative to TLC's explicit-state model checking.
2. The docs state Apalache translates verification problems to logical constraints solved by an SMT solver such as Microsoft Z3.
3. The running docs describe `check` as performing operations including typechecking and bounded model checking; by default it encodes a symbolic execution of length `k` and a violation of an invariant into SMT.
4. The principles docs state Apalache requires type annotations and has writing principles for taking advantage of symbolic model checking.
5. The project page states Apalache supports bounded model checking and inductiveness checking for TLA+ specifications.
