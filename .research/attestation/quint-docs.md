# Quint docs attestation

1. Quint docs describe Quint as a specification language for models and properties, with simulator and model checker support; violated properties produce counterexample traces.
2. Quint model-checker docs describe TLC as an available backend for `quint verify --backend=tlc`, with TLC as an explicit-state checker.
3. Quint getting-started docs state the simulator can miss executions, while model checking via `verify` increases assurance; the model checker requires JDK >=17.
4. Quint language docs state Quint is used to define state machines and properties, not regular executable code, and is based on TLA+ semantics.
5. Quint GitHub/docs describe it as a modern specification language especially suited to distributed systems, combining TLA foundations with type checking and development tooling.
